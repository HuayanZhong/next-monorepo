## 文档导航

开始工作前，先阅读以下文件了解项目上下文：

- **项目概览**: 根目录 `README.md` — 项目是什么、为什么存在、技术栈
- **Agent 指令**: 根目录 `AGENTS.md` — Next.js 版本注意事项
- **Git 规范**: `.trae/rules/git-commit-message.md` — 提交消息格式
- **前端规范**: `.trae/rules/frontend/` — 按领域拆分的子规则目录
  - `react-best-practices.md` — React 组件模式
  - `css-naming.md` — 样式与类名约定

## 工作原则

- 动手前先读相关代码和文档，不要凭猜测写代码
- 遇到不确定的 API 或约定，先查项目内的实际用法
- 优先复用已有模式，而非引入新抽象
- 改动要最小化，聚焦于当前任务
